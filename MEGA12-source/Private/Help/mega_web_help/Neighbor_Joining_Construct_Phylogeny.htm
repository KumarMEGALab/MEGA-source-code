Neighbor Joining (Construct Phylogeny)

Phylogeny | Construct/Test Neighbor-Joining Tree…

This command is used to construct (or Test) a neighbor-joining (NJ) tree (Saitou & Nei 1987Saitou_and_Nei_1987).  The NJ method is a simplified version of the minimum evolution (ME) method, which uses distance measures to correct for multiple hits at the same sites, and chooses a topology The branching pattern of a tree is its topology. showing the smallest value of the sum of all branches as an estimate of the correct tree.  However, the construction of an ME tree is time-consuming because, in principle, the S values for all topologies have to be evaluated and the number of possible topologies (unrooted trees) rapidly increases with the number of taxa A taxon is the individual unit whose evolutionary relationship is being investigated. Depending on the study, “taxa” may refer to species, populations, individuals, or sequences within an individual..

With the NJ method, the S value is not computed for all or many topologies. The examination of different topologies is imbedded in the algorithm, so that only one final tree is produced. This method does not require the assumption of a constant rate of evolution so it produces an unrooted tree An unrooted tree is one in which no assumption is made regarding the ancestor of all the taxa in the tree..  However, for ease of inspection, MEGA displays NJ trees in a manner similar to rooted trees. The algorithm of the NJ method is somewhat complicated and is explained in detail in Nei and Kumar (2000Nei_and_Kumar_2000). 

For constructing the NJ tree, MEGA may request that you specify the distance estimation method, subset of sites to include, and whether to conduct a test of the inferred tree A tree reconstructed from the observed sequence or other appropriate data using any tree-making method (such as UPGMA, NJ, ME, or MP) is known as an inferred or reconstructed tree.RH_Inferred_tree through an Analysis PreferencesHC_NJ_Analysis_Options dialog box.
